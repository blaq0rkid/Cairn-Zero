### 7. ACCESSIBILITY & COMPLIANCE (THE "ETHOS OF CERTAINTY")

#### 7.1 Digital Accessibility Standards
To align with our commitment to certainty and our physical Braille-integrated hardware, the website must meet high accessibility standards without sacrificing the sleek, minimalist aesthetic.

*   **WCAG 2.1 Level AA Compliance:** The baseline for all digital elements.
*   **Visual Clarity:** Use high-contrast color ratios (minimum 4.5:1 for text) within the Black/White/Stone palette.
*   **Typography:** Use simple, sans-serif fonts (e.g., Inter, Roboto, or similar) with generous line spacing and clear hierarchy for readability.
*   **Keyboard Navigation:** Full site functionality must be accessible via keyboard only (no "mouse traps").
*   **Screen Reader Optimization:**
    *   Semantic HTML structure (proper use of H1, H2, H3 tags).
    *   Descriptive Alt-text for all images (especially the Cairn logo and hardware visuals).
    *   ARIA labels for all interactive elements and buttons.
*   **Mobile Responsiveness:** A "mobile-first" approach. The layout must be perfectly fluid and readable on all devices, ensuring the "Succession Bridge" is accessible from anywhere.

#### 7.2 Legal & Regulatory Compliance
*   **Privacy by Design:** The site must reflect our "Zero-Knowledge Sovereignty." No unnecessary tracking scripts or third-party cookies that compromise user privacy.
*   **Consent Management:** Clear, non-intrusive cookie/privacy consent that aligns with our professional tone.
*   **Terms & Disclosures:** All legal links in the footer must be clearly labeled and easy to find, reinforcing the "Certainty" we provide.

#### 7.3 The "Sleek & Simple" Filter
*   **Zero Bloat:** No heavy animations or "fluff" that slows down the site or distracts from the mission.
*   **Clear Paths:** Every page element should guide the user toward either understanding their "Systemic Risk" or securing their "Succession Bridge."
*   **Minimalist Forms:** Onboarding and contact forms should be short, asking only for what is absolutely necessary.

### 8. USER PORTALS: OWNER VS. SUCCESSOR

The website must support two distinct, password-protected (or hardware-authenticated) environments that manage the "Active/Archive" states.

#### 8.1 The Founder Dashboard (The "Active" Zone)
*   **Purpose:** Where the founder sets up their "Succession Bridge."
*   **Key Features:**
*   **Cairn ID Management:** Link and manage the 6-character CZ-XXXX IDs for physical hardware.
*   **Continuity Trigger Configuration:** Set up "Succession Pings" (email/SMS checks).
*   **Succession Logic:** Define the sequence (e.g., Successor A, then Successor B if A is unavailable).
*   **Systemic Risk Audit:** A dashboard tool to track and consolidate "Digital Sprawl" (passwords, accounts).
*   **Authentication:** FIDO2/YubiKey for Cairn Lite; PIN-Pad Hardware for Founder Guard.

#### 8.2 The Successor Portal (The "Archive" Zone)
*   **URL:** `mycairnzero.com/successor` (This URL is laser-etched on physical keys).
*   **Purpose:** A secure, zero-knowledge entry point for designated heirs/partners.
*   **Key Features:**
*   **Access Key Entry:** A high-visibility field to enter the 6-character Cairn ID (CZ-XXXX) found on the physical device.
*   **The "Certainty" Handover:** This portal remains "locked" until the founder’s triggers are officially tripped (the Succession Gap occurs).
*   **Information Retrieval:** Once unlocked, it provides the "Keys to the Kingdom"—decryption instructions and access to indexed business data.
*   **Zero-Knowledge Ethos:** The system must clearly state that Cairn Zero has zero access to this data; we are simply the bridge.

#### 8.3 Branding the Split
*   **Visual Distinction:** The Founder Dashboard should feel like a "Control Center" (Active). The Successor Portal should feel like a "Vault" (Archive).
*   **Instructional Clarity:** Both portals must have extremely clear, simple instructions, as the Successor may be accessing the site during a high-stress emergency.

### 9. TECHNICAL DEPLOYMENT (NETLIFY & HOSTING)

The site is designed to be hosted on **Netlify**, which aligns with our "Certainty-Only" ethos by providing speed, security, and 100% uptime.

#### 9.1 Why Netlify?
*   **Speed (Edge Network):** The Successor Portal must load instantly. Netlify’s global edge network ensures there is no latency for someone in an emergency.
*   **Security (SSL/HTTPS):** Automatic SSL encryption for every page, reinforcing our "Secure Bridge" messaging.
*   **Continuous Deployment (CI/CD):** Claude will push the code to a Git repository (e.g., GitHub), and Netlify will automatically build and deploy the site whenever we make an update. This means zero downtime during maintenance.
*   **Serverless Functions:** Netlify Functions may be used to handle the "Succession Pings" and trigger logic without requiring a complex, vulnerable server backend.

#### 9.2 Claude AI's Role in Deployment
*   **Static Site Generation:** Claude should build the site using a modern framework (e.g., React or Next.js) that exports to static files suitable for Netlify hosting.
*   **Netlify Forms:** Any contact or "Systemic Risk Audit" forms should use Netlify Forms for secure, serverless data handling.
*   **Redirects & Pretty URLs:** Claude must configure `_redirects` to ensure `mycairnzero.com/successor` and other paths work correctly.

#### 9.3 The "Day 1" Guarantee
*   Netlify supports "Preview Deploys." Before the site goes live, Cairn Zero must be able to review a private version of exactly how it will look and behave to confirm there are no broken links or misstatements.
